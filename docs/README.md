# SwiftII documentation

This directory holds all SwiftII documentation, organized by **who it's
for**. The project intro, build instructions, and current status live in the
[top-level README](../README.md); start there if you landed here from
elsewhere.

```
docs/
├── using/         ← people USING SwiftII (write & run programs)
├── contributing/  ← people WORKING ON SwiftII (the codebase)
│   └── design/    ← numbered design-decision records
├── testing/       ← TESTERS (test layers + on-target playbooks)
└── screenshots/   ← generated Apple II screen captures used in the docs
```

---

## using/ - writing SwiftII programs

For anyone writing Swift to run on SwiftII. No knowledge of the C codebase
needed.

- [TUTORIAL.md](using/TUTORIAL.md) - hands-on: boot a disk, use the REPL, write and run a program, graphics, the compiler.
- [CHEATSHEET.md](using/CHEATSHEET.md) - one-page quick reference: syntax, types, every built-in with its platform tag.
- [API.md](using/API.md) - developer API reference: each built-in / type with signature, platform availability, and an example.
- [LANGUAGE.md](using/LANGUAGE.md) - the complete language spec (semantics, grammar, differences from Swift).
- [FEATURES.md](using/FEATURES.md) - every feature and what it costs (disk + memory).

## contributing/ - working on the codebase

For anyone changing SwiftII itself. Suggested first read for a new
contributor: **[DEVELOPING.md](contributing/DEVELOPING.md)**, then
**[ARCHITECTURE.md](contributing/ARCHITECTURE.md)**.

- [ARCHITECTURE.md](contributing/ARCHITECTURE.md) - **how it all fits together**: the pipeline, the value/memory model, the two binary families, the build matrix. Start here for the mental model.
- [DEVELOPING.md](contributing/DEVELOPING.md) - new-contributor getting-started guide and the inner dev loop.
- [BUILDING.md](contributing/BUILDING.md) - toolchain setup and build targets.
- [CONSTRAINTS.md](contributing/CONSTRAINTS.md) - hard platform/toolchain limits (read before writing any code).
- [STYLE.md](contributing/STYLE.md) - coding style (C90/C17, naming, comments).
- [PROJECT_LAYOUT.md](contributing/PROJECT_LAYOUT.md) - the full repo tree and rationale.
- [MEMORY_MAP.md](contributing/MEMORY_MAP.md) - zero-page, RAM, and language-card layout. Update it when you allocate zero-page.
- [OPCODES.md](contributing/OPCODES.md) - the bytecode opcode reference.
- [SWB.md](contributing/SWB.md) - the `.swb` compiled-program file format (Compiler → Runner hand-off).
- [ROADMAP.md](contributing/ROADMAP.md) - what to work on next · [ROADMAP-MAYBE.md](contributing/ROADMAP-MAYBE.md) - deferred / maybe-never ideas.
- [LESSONS.md](contributing/LESSONS.md) - running log of hard-won gotchas. Skim before any non-trivial change.

**Architectural changes**: write a design doc under
[design/](contributing/design) using the
[template](contributing/design/001-template.md). A few entry points:

- [002 - Heap and strings](contributing/design/002-heap-and-strings.md)
- [003 - Apple II Plus source-level digraphs](contributing/design/003-apple2-input-method.md)
- [011 - Extras code in the Saturn/aux bank (XLC)](contributing/design/011-extras-lc-in-saturn-aux.md)
- [015 - Bigger programs: the Family B Compiler/Runner](contributing/design/015-bigger-programs-pascal-toolchain.md)
